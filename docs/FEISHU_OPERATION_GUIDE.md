# 飞书侧操作流程
**版本：** v0.1.0 | **更新时间：** 2026-04-27 | **内容：** 飞书群机器人接入、消息格式、后台绑定与日常操作说明

## 1. 创建并配置飞书群机器人
1. 在目标群聊中添加自定义机器人。
2. 记录机器人的 Webhook 地址。
3. 在部署环境配置 FEISHU_WEBHOOK_URL（用于系统主动通知）。
4. 在飞书开放平台配置事件订阅回调地址：
   - 回调 URL: http://你的服务地址/api/feishu/webhook
   - 开启群消息事件（文本消息）
5. 安全配置（推荐必须开启）：
   - FEISHU_VERIFICATION_TOKEN：填飞书事件订阅的 Verification Token
   - FEISHU_SIGNING_SECRET：填飞书事件订阅的 Signing Secret

## 2. 后台初始化与账号绑定
1. 访问前端页面，先执行管理员初始化。
2. 管理员登录后创建小孩、创建操作用户并分配可控制小孩。
3. 每个用户在后台“系统配置”中绑定自己的飞书 OpenID。
4. 管理员可在“操作用户管理”中直接维护操作员的飞书 OpenID。

## 3. 群聊支持的指令/语句
以下消息会触发识别（文本消息）：
1. 调整每日额度
   - 示例：设置小明每日零花钱 12 元
2. 设置额外奖励项目
   - 示例：设置小明奖励项目家务 5 元
3. 扣除消费
   - 示例：扣除小明 8 元 买零食
   - 示例：消费 -5 元（等效于反向增加 5 元）
4. 设置每周通知时间
   - 示例：设置每周统计通知 20:30
5. 完成奖励触发
   - 示例：小明完成家务

## 4. 主动通知类型
机器人会向群里主动推送卡片消息：
1. 金额变动通知
   - 增减金额
   - 变动原因
   - 操作来源（管理员/操作员/机器人/定时任务）
2. 系统操作反馈
   - 每日额度调整结果
   - 奖励项目设置结果
   - 周报通知时间设置结果
3. 每周统计通知（周一）
   - 上周总增加金额
   - 上周消费金额
   - 当前剩余金额
   - 最大消费项目和金额
   - AI/规则建议

## 5. 机器人消息过滤策略
系统会自动忽略：
1. sender_type = bot 的消息
2. 系统配置 ignoreBotUserIds 中的发送者
3. 未绑定飞书账号的发送者

## 6. 推荐运营流程
1. 管理员每周检查一次奖励规则和每日额度。
2. 操作用户只管理自己负责的小孩。
3. 群内尽量使用“姓名 + 动作 + 金额/项目”的句式，识别准确率更高。
