# 移动端页面布局规范
**版本：** v0.1.0 | **更新时间：** 2026-04-27 | **内容：** 基于参考项目提炼的移动端后台布局规范

## 概述
本项目主要面向手机端，PC 仅作为访问终端，不做桌面化重排。

## 1. 视口策略
1. 页面容器使用固定上限宽度 430px。
2. 桌面端访问时页面居中显示，仍保持手机布局层次。
3. 所有核心操作按钮需在一屏内可触达。

## 2. 视觉令牌
1. 主色：橙色系（用于强调操作）
2. 辅色：青绿色（用于状态反馈）
3. 中性色：浅米色 + 浅蓝渐变背景
4. 卡片：半透明白底 + 轻阴影 + 圆角

## 3. 交互节奏
1. 页面入场动画 300ms~450ms。
2. 所有反馈使用卡片或提示条，不使用浏览器原生 alert。
3. 关键操作（金额变动、规则设置）在操作完成后立即给出文本反馈。

## 4. 布局分区
1. 顶部：系统名称 + 当前账号 + 退出入口
2. 中段：小孩管理与金额操作
3. 下段：系统配置、流水、周报

## 5. 表单规范
1. 表单输入区使用 12px 圆角和统一边框。
2. 单行操作优先；多字段场景使用分组卡片。
3. 管理员高风险操作需在文案上明确用途（例如绑定 OpenID）。
